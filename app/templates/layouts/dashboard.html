Docker Configuration Warning

{% if mount_warning.epubs_only %}

Your Docker container is missing the /epubs bind mount. Without this, your downloaded EPUB files will be lost when the container is updated or recreated.

{% else %}

Your Docker container is missing bind mounts for data persistence. Without these, your converted books and covers will be lost when the container is updated or recreated.

{% endif %}
How to fix this
{% if mount_warning.epubs_only %}

Add the /epubs volume mount (required)

Add this volume mount to your docker-compose.yml:

volumes:
  - ./epubs:/litkeeper/app/data/epubs

After updating, restart your container: docker compose down && docker compose up -d

{% else %}

Add volume mounts to persist your data

Add these volume mounts to your docker-compose.yml:

volumes:
  - ./epubs:/litkeeper/app/data/epubs      # Required for downloads
  - ./html:/litkeeper/app/data/html        # Required for library
  - ./covers:/litkeeper/app/data/covers    # Required for library
  - ./logs:/litkeeper/app/data/logs        # Optional: only needed if logging is enabled

After updating, restart your container: docker compose down && docker compose up -d

Alternative: Disable library features

If you only need downloads and don't want the library, set ENABLE_LIBRARY=false in your environment:

environment:
  - ENABLE_LIBRARY=false

Note: You still need the /epubs mount to persist downloads.

{% endif %}
